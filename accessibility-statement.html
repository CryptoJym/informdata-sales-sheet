Our Commitment

Vuplicity is committed to ensuring digital accessibility for all users, including people with disabilities. We continually work to improve the accessibility and usability of our platform to ensure everyone can access our background screening services effectively.

1. Accessibility Standards

We strive to conform to the Web Content Accessibility Guidelines (WCAG) 2.1 at the AA level. These guidelines explain how to make web content more accessible for people with disabilities and more user-friendly for everyone.

1.1 WCAG Principles

Our accessibility efforts are guided by the four core principles of WCAG:

Perceivable

Information and user interface components must be presentable to users in ways they can perceive.

  • Text alternatives for images
  • Captions and transcripts for multimedia
  • Content adaptable to different presentations
  • Sufficient color contrast

Operable

User interface components and navigation must be operable by all users.

  • Full keyboard accessibility
  • Sufficient time to complete tasks
  • No content that causes seizures
  • Clear navigation and wayfinding

Understandable

Information and operation of the user interface must be understandable.

  • Readable and predictable text
  • Consistent navigation patterns
  • Clear error identification and recovery
  • Helpful input assistance

Robust

Content must be robust enough for interpretation by a wide variety of user agents, including assistive technologies.

  • Valid, semantic HTML
  • Proper ARIA attributes
  • Compatible with assistive technologies
  • Future-proof code

2. Accessibility Features

2.1 Navigation and Structure

  • Skip Navigation Links: "Skip to main content" links allow keyboard and screen reader users to bypass repetitive navigation
  • Semantic HTML: Proper use of headings (h1-h6), landmarks, and structural elements for clear document outline
  • Consistent Layout: Predictable navigation and page structure across the platform
  • Keyboard Navigation: All interactive elements accessible via keyboard (Tab, Enter, Arrow keys)
  • Focus Indicators: Clear visual focus states for keyboard navigation

2.2 Visual Design

  • Color Contrast: Minimum 4.5:1 contrast ratio for normal text, 3:1 for large text (WCAG AA)
  • Color Independence: Information conveyed through color is also indicated by text or patterns
  • Resizable Text: Text can be resized up to 200% without loss of functionality
  • Responsive Design: Content adapts to different screen sizes and orientations
  • Dark Mode Support: System preference respecting for reduced eye strain

2.3 Screen Reader Support

  • Alt Text: Descriptive alternative text for all informative images
  • ARIA Labels: Appropriate ARIA attributes for dynamic content and complex widgets
  • Form Labels: Properly associated labels for all form inputs
  • Status Messages: Live regions announce dynamic content changes
  • Table Headers: Data tables include proper headers and scope attributes

2.4 Forms and Input

  • Clear Labels: All form fields have visible, descriptive labels
  • Error Identification: Errors clearly identified with descriptive messages
  • Input Assistance: Helpful hints and format examples provided
  • Required Fields: Clearly marked with both visual and programmatic indicators
  • Validation Feedback: Real-time validation with accessible error messages

2.5 Multimedia

  • Video Captions: Closed captions provided for all video content
  • Transcripts: Text transcripts available for audio and video content
  • Audio Descriptions: Descriptive audio tracks for visual content where applicable
  • Media Controls: Accessible play/pause, volume, and playback controls

5. Known Limitations

While we strive for full accessibility, we acknowledge the following current limitations:

5.1 Third-Party Content

  • PDF Reports: Some legacy PDF reports may not be fully accessible. We are working to remediate these documents and provide accessible alternatives upon request.
  • External Links: We cannot guarantee the accessibility of third-party websites linked from our platform.
  • Embedded Content: Some embedded widgets (e.g., payment processors, chat support) are provided by third parties and may have accessibility gaps.

5.2 Complex Data Visualizations

  • Charts and graphs include text alternatives, but may not provide the full interactive experience for screen reader users
  • We are actively working on improving data table alternatives for complex visualizations

5.3 Legacy Features

  • Some older features of the platform are undergoing accessibility remediation
  • We prioritize new development to be fully accessible from launch

6. Ongoing Efforts

Accessibility is an ongoing commitment. Our current initiatives include:

6.1 Regular Audits

  • Quarterly automated accessibility scans using industry-standard tools (axe, WAVE)
  • Annual comprehensive manual audits by accessibility specialists
  • Bi-annual testing with users who rely on assistive technologies
  • Continuous monitoring of WCAG guideline updates

6.2 Team Training

  • Regular accessibility training for all developers, designers, and content creators
  • Accessibility champions embedded in each product team
  • Accessibility considerations in code reviews and QA processes

6.3 User Feedback Integration

  • Dedicated accessibility feedback channel
  • Quarterly review of accessibility-related support tickets
  • User research sessions with people with disabilities
  • Accessibility issues prioritized in product roadmap
