General API – Requests & Responses

Organized reference of request/response JSON samples by product family. Use these as payload guides alongside tenant‑specific endpoint paths. For MVR details and end‑to‑end flow, see the MVR API page.

How to Use These Samples

  • Endpoints are tenant‑specific (e.g., POST /api/orders); confirm with InformData which path and auth your account uses.
  • Requests share a common structure: order → subject, product, jurisdiction.
  • Responses include order.id, status, pricing, and result sections (e.g., cases, otherResults).

Raw JSON files are included under integrations/client_api/ in this repo and served by the site for download.

Endpoint Conventions (Placeholder)

  • POST /api/orders – create order for a given product category.
  • GET /api/orders/{orderId} – check status.
  • GET /api/reports/{reportId} – fetch normalized report + documents.
